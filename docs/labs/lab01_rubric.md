# Lab 1: Portfolio Setup - Evaluation Rubric

## Overview
This rubric uses an ungrading approach to help you understand expectations and assess your own work. Focus on learning the tools, following technical steps accurately, and building a foundation for the semester.

---

## 1. Tool Setup & Account Creation

### Exceeds Expectations
- Successfully sets up all three required tools (GitHub, GitHub Desktop, VS Code)
- GitHub username is professional and appropriate for portfolio sharing
- Configures optional but recommended settings (e.g., VS Code integration in GitHub Desktop)
- Demonstrates understanding of why each tool is used and how they work together
- Proactively troubleshoots any installation issues

### Meets Expectations
- Successfully creates GitHub account with appropriate username
- Installs and configures GitHub Desktop correctly
- Installs and opens VS Code successfully
- All tools are functional and properly linked
- Follows setup instructions accurately

### Does Not Meet Expectations
- Missing one or more required tools
- GitHub username is unprofessional or inappropriate
- Installation incomplete or tools not properly configured
- Unable to demonstrate that tools are working
- Did not follow setup instructions

---

## 2. Repository Creation & Management

### Exceeds Expectations
- Creates repository from template with thoughtful naming and description
- Understands the purpose and structure of the repository
- Successfully clones repository to appropriate local location
- Demonstrates understanding of the relationship between local and remote repositories
- Shows good organizational habits in file management

### Meets Expectations
- Successfully creates repository using "Use this template" feature
- Repository is named appropriately and set to public
- Successfully clones repository to computer using GitHub Desktop
- Can locate and navigate repository files on local machine
- Repository structure remains intact

### Does Not Meet Expectations
- Repository creation incomplete or unsuccessful
- Repository set to private (preventing GitHub Pages deployment)
- Failed to clone repository or cannot locate files locally
- Repository structure is broken or files are missing
- Unable to demonstrate basic repository navigation

---

## 3. GitHub Pages Deployment

### Exceeds Expectations
- Correctly enables GitHub Pages on first attempt
- Understands deployment process and can explain it
- Proactively checks deployment status and troubleshoots issues
- Verifies site is live and functioning correctly
- Understands the relationship between pushes and site updates

### Meets Expectations
- Successfully enables GitHub Pages with correct settings
- Site deploys and is accessible at correct URL
- Can demonstrate that portfolio is live on the web
- Understands that changes require commits and pushes to appear online

### Does Not Meet Expectations
- GitHub Pages not enabled or configured incorrectly
- Site does not deploy or is inaccessible
- Unable to locate or share live URL
- Does not understand deployment process
- Cannot demonstrate functional website

---

## 4. Template Exploration & Understanding

### Exceeds Expectations
- Thoroughly explores and understands all template files
- Can explain the purpose of each file (HTML, CSS, images, README)
- Understands the structure and organization of the template
- Makes connections between HTML structure and what appears in browser
- Asks insightful questions about code or design choices

### Meets Expectations
- Opens and reviews key template files (index.html, styles.css)
- Understands basic file structure and purpose
- Can identify where content appears on the webpage
- Recognizes which files to edit vs. which to leave alone
- Follows exploration instructions in lab document

### Does Not Meet Expectations
- Does not explore template files before editing
- Cannot identify basic file purposes
- Shows no understanding of template structure
- Makes changes without understanding what they affect
- Skips exploration phase entirely

---

## 5. Personalization & Customization

### Exceeds Expectations
- Makes thoughtful, meaningful personalizations throughout portfolio
- Updates all placeholder content with accurate personal information
- Adds personal style through color schemes, fonts, or layout adjustments
- Content is polished, professional, and error-free
- Demonstrates creativity while maintaining functionality
- Makes additional improvements beyond minimum requirements

### Meets Expectations
- Updates name, bio, and contact information accurately
- Personalizes at least one section (about, projects, or contact)
- Makes at least one visual customization (colors, fonts, or images)
- Content is clear and professional
- All links work correctly
- Changes are intentional and improve the site

### Does Not Meet Expectations
- Minimal or no personalization beyond changing name
- Placeholder text remains throughout site
- No visual customization attempted
- Content contains errors or is unprofessional
- Broken links or non-functional elements
- Changes appear random or break the site

---

## 6. Version Control Practice (Commit & Push)

### Exceeds Expectations
- Makes multiple logical commits with clear, descriptive messages
- Commits represent meaningful chunks of work
- Pushes changes successfully and verifies deployment
- Understands the purpose and importance of version control
- Demonstrates good version control habits (commit frequency, message quality)

### Meets Expectations
- Successfully commits changes to local repository
- Commit messages are present and somewhat descriptive
- Successfully pushes changes to GitHub
- Live site updates with changes
- Demonstrates basic understanding of commit/push workflow

### Does Not Meet Expectations
- Changes not committed or commit messages missing/unclear
- Unable to push changes to GitHub
- Live site does not reflect changes
- Does not understand commit/push process
- Significant errors in version control workflow

---

## 7. Reflection & Metacognition

### Exceeds Expectations
- Provides thoughtful, substantive reflection on learning process
- Identifies specific challenges and how they were overcome
- Makes connections between technical skills and course goals
- Articulates understanding of portfolio's role in the semester
- Reflects on growth mindset and learning strategies
- Shows awareness of what still needs improvement

### Meets Expectations
- Completes reflection with honest assessment of experience
- Identifies at least one challenge and one success
- Demonstrates engagement with the learning process
- Reflection shows genuine thought rather than rushing through
- Addresses prompt questions adequately

### Does Not Meet Expectations
- Reflection is missing, incomplete, or superficial
- Shows no evidence of critical thinking about the process
- Does not address prompt questions
- Reflection is perfunctory or appears rushed
- No evidence of learning or growth mindset

---

## 8. Deliverables & Submission

### Exceeds Expectations
- Submits all required elements on time
- GitHub repository is well-organized and professional
- Live site is polished and fully functional
- All links work and point to correct locations
- Goes beyond minimum requirements
- Shows attention to detail in all aspects

### Meets Expectations
- Submits all required deliverables:
  - Link to GitHub repository
  - Link to live GitHub Pages site
  - Both links work correctly
- Repository contains all required files
- Site is functional and displays correctly
- Submitted by deadline

### Does Not Meet Expectations
- Missing required deliverables
- Links don't work or point to wrong locations
- Repository is incomplete or inaccessible
- Site is non-functional or has major issues
- Late submission without prior arrangement
- Submission is disorganized or unprofessional

---

## Self-Assessment Questions

As you complete this assignment, consider:

1. **Technical Competence**: Can I successfully use GitHub, GitHub Desktop, and VS Code? What do I still need to practice?

2. **Problem Solving**: When I encountered issues, how did I troubleshoot? What resources did I use?

3. **Attention to Detail**: Did I follow instructions carefully? Are all my links working and content accurate?

4. **Professionalism**: Would I be comfortable sharing this portfolio with a potential employer?

5. **Understanding**: Do I understand *why* we're using these tools, not just *how* to use them?

6. **Growth Areas**: What aspects of this lab were most challenging? What do I need to revisit?

---

## Notes on Ungrading Approach

This rubric is designed to help you:
- Understand what successful technical setup and portfolio creation looks like
- Self-assess your readiness to build on this foundation throughout the semester
- Identify areas where you need additional support or practice
- Take ownership of your learning and skill development

Remember: This lab establishes the infrastructure for the entire semester. Solid work here makes everything else easier. The goal is not perfection, but functional tools and understanding of the workflow.
