Vibe Coding

AI-Assisted Development

Overview

"Vibe coding" refers to AI-assisted code generation where you describe what you want in natural language and the AI produces working code. By now, you've learned HTML/CSS fundamentals (Lab 1), explored AI capabilities critically (Lab 2), and built analysis and visualization skills (Labs 3-4).

In this lab, you'll explore AI as a coding collaborator — learning when it's helpful, when it's problematic, and how to maintain your understanding while leveraging AI assistance.

Key Principle: You must be able to read, explain, and modify any code you use, regardless of who (or what) wrote it.

Part 1: The AI Coding Spectrum

AI coding tools exist on a spectrum:

Approach Human Role AI Role Best For
Fundamentals First Write all code; AI explains errors Teacher/debugger Learning new concepts
Collaborative Design logic; AI implements Co-pilot Familiar patterns, new syntax
Vibe Coding Describe outcome; AI generates Generator Rapid prototyping, repetitive tasks
Full Automation Minimal involvement Black box ⚠️ Risky - no understanding
Remember: You must be able to read, explain, and modify any code you use.

Part 2: Experiments in AI-Assisted Development

Complete at least two of the following experiments:

Experiment A: Portfolio Enhancement

Use AI to add a feature to your Skills Portfolio.

Options:

  • Responsive navigation menu (hamburger menu on mobile)
  • Dark mode toggle
  • Image gallery or carousel
  • CSS animations or transitions
  • Accessibility improvements (ARIA labels, keyboard navigation)

Process:

  1. Plan first — sketch what you want, identify the components
  2. Prompt the AI — describe the feature clearly
  3. Review the code — read every line; add comments explaining what each part does
  4. Test thoroughly — does it work? Is it accessible? Does it break anything?
  5. Refine — ask AI to modify, or make changes yourself

Experiment B: Python Code Generation

Use AI to generate Python code for a data analysis task.

Options:

  • Clean/transform a messy dataset
  • Create a specific type of visualization
  • Perform statistical analysis
  • Scrape data from a website
  • Process text or perform sentiment analysis

Process:

  1. Define the task — be specific about inputs, outputs, and requirements
  2. Generate code — prompt AI with clear instructions
  3. Review line-by-line — add comments explaining the logic
  4. Test with sample data — does it produce expected results?
  5. Handle edge cases — what breaks it? Ask AI to fix, or fix yourself

Experiment C: Debug with AI

Take broken code and use AI to fix it.

Options:

  • Find a buggy code snippet online (or write intentionally broken code)
  • Use AI to identify and fix the issue
  • Compare AI's solution to your own debugging process

Process:

  1. Describe the problem to the AI
  2. Evaluate the fix — does it actually solve the problem?
  3. Understand the root cause — why was the code broken?
  4. Document — what did you learn about debugging with AI?
Tip: Document your prompts and AI responses as you go — you'll need them for your reflection and submission.

Part 3: Critical Reflection

Write a 400-600 word reflection addressing the following prompts:

Technical Understanding

  1. What code did the AI generate? Describe the feature/function in your own words.
  2. How does it work? Explain the key logic or mechanisms. If you can't explain it, that's a red flag.
  3. What did you modify? Did you need to fix bugs, adjust styling, or change functionality? Why?

Process Evaluation

  1. When was AI helpful? What tasks did it accelerate or simplify?
  2. When was AI problematic? Where did it produce incorrect, inefficient, or unnecessarily complex code?
  3. What's your role? In an AI-assisted workflow, what remains uniquely human? What can't (or shouldn't) be automated?

Ethical & Pedagogical Considerations

  1. Learning trade-offs: Does vibe coding help or hinder your development as a programmer? Under what conditions?
  2. Attribution: When using AI-generated code, what does proper citation look like? (Consider academic vs. professional contexts)
  3. Your guidelines: What personal rules will you follow for using AI in coding? When will you use it? When won't you?
Important: Do not use AI to write your reflection. This must be your own critical thinking and personal voice.

Part 4: Portfolio Integration

Add a Lab 5 entry to your Skills Portfolio that includes:

  • Description of what you built/analyzed with AI assistance
  • Key code snippet with detailed comments
  • Brief discussion of your vibe coding guidelines
  • Screenshot or demo of the working feature
Note: If you used AI to enhance your portfolio (Experiment A), your Lab 5 entry should explain the new feature and your process.

Create a new page called lab05.html in your portfolio that showcases your work. Include appropriate navigation links so visitors can find it.

Self-Assessment

Before submitting, take a moment to reflect on your work honestly:

Questions to Consider

  • Know What You Know: Can I separate what I understand from what I don't? It's fine to say "I understand the logic but not the syntax" — it's not fine to pretend I understand everything when I don't.
  • The Explanation Test: If I had to explain my code to a classmate at my same skill level, could I walk them through what it does and why — even if I couldn't write it from scratch?
  • Modification Honesty: Did I actually change the AI's output, or did I accept it wholesale? If I changed nothing, was the code genuinely perfect — or did I not look closely enough?
  • Choosing Your Level: Did I pick experiments that challenged me at my current skill level, or did I stay in my comfort zone? A beginner tackling something unfamiliar is doing harder work than an experienced coder generating something routine.
  • Reflection Authenticity: Is my reflection my own thinking, or did I write what I thought the instructor wanted to hear? Do my "personal guidelines" for AI use reflect how I'll actually behave, or are they performative?
  • Credit Clarity: If a classmate read my portfolio page, would they understand exactly what the AI contributed, what I contributed, and what I'm still figuring out?

Use these questions to guide final revisions before submission. Honest self-assessment helps you identify areas of strength and opportunities for growth.

Submission

Guidelines for AI Use in This Lab

✅ Encouraged:
  • Using AI to generate initial code
  • Asking AI to explain code concepts
  • Requesting debugging help
  • Exploring multiple AI-generated approaches
❌ Not Allowed:
  • Submitting AI code you don't understand
  • Skipping the commenting/documentation process
  • Using AI to write your reflection
  • Copying code without attribution

Looking Ahead

The skills you develop here — prompting effectively, evaluating AI output, maintaining code literacy — will serve you in Lab 6 (Python mapping) and your Semester Project.

The question isn't "Should I use AI?" but "How do I use AI responsibly and effectively?"

Remember: The point isn't to avoid AI, but to use it thoughtfully while maintaining your understanding and agency.
