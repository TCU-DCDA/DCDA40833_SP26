GitHub as Infrastructure

Version Control • Backup • Hosting

Infrastructure of Code

Code becomes Culture

How does code become culture? Through the infrastructure that stores, tracks, and publishes it.

The Problem We're Solving

❌ Before GitHub

  • Lost files ("my laptop died")
  • Version chaos (final_v2_REAL_final.html)
  • No collaboration history
  • Manual file hosting
  • Breaking changes with no undo

✅ With GitHub

  • Cloud backup automatically
  • Time travel (undo any change)
  • Perfect history of who/what/when
  • Free hosting included
  • Safety net for experimentation

Role #1: Version Control

Imagine a save system for your code that lets you time travel.

Key Concept: Every "Commit" is a checkpoint you can return to later. It's like a save point in a video game.

Role #2: Backup

💻 Your Computer

This is your "Local" environment. It's where you type code. It is fragile (coffee spills, disk failures).

☁️ GitHub Cloud

This is "Remote". It is resilient. Once you Push your code here, it is safe.

Role #3: Web Hosting (GitHub Pages)

We don't just want to store code; we want to publish a website. GitHub Pages builds your site automatically from your repository.

Your Repo (HTML/CSS) --> GitHub Build Process --> https://you.github.io

Role #4: Collaboration

GitHub allows multiple people to work on the same project without overwriting each other's work.

Transparency

Open Source means the history of every contribution is visible. You can see exactly who wrote what line of code and when.

The Complete Workflow

This is the cycle you will repeat for every lab and project in this course.

1. Create Locally

Write your HTML/CSS in VS Code.

2. Commit

Save your changes to your local git history with a message.

3. Push

Upload your creates to GitHub.com (Backup).

4. Deploy

GitHub Pages automatically updates your live site.
